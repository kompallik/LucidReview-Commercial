The criteria source matters for this market.

For Medicare Advantage, CMS guidance explicitly requires decisions to align with Traditional Medicare criteria (NCDs/LCDs). MCG and InterQual are only permitted where CMS criteria are "not fully established" — making public CMS criteria the legally correct foundation.

LucidReview ingests CMS criteria automatically via the Coverage API, compiles them into executable CQL, and evaluates them deterministically against FHIR-normalized patient data. No licensing dependency. Full regulatory alignment.

The AI Agent Pipeline

8 steps from case to determination

Claude AI orchestrates the entire review workflow through 14 MCP tools. Deterministic CQL criteria decide the outcome. The LLM synthesizes and explains.

🔍
1. Gather Case Data
Fetches case metadata, ICD-10/CPT codes, clinical info, and member coverage verification from the UM system via REST APIs.
um_get_caseum_get_clinical_info
📄
2. Extract Documents
Downloads clinical PDFs and extracts text using pdf-parse with AWS Textract OCR fallback for scanned documents.
pdf_extract_textOCR fallback
🧠
3. Clinical NLP
cTAKES/medspaCy extracts clinical entities with assertion status (affirmed/negated/uncertain) and temporality for each finding.
nlp_extractcTAKESAssertion
⚕️
4. FHIR Normalization
Maps all evidence to FHIR R4 resources (Patient, Condition, Observation, Procedure) with provenance tracking. Posted to HAPI FHIR server.
fhir_normalizeHAPI FHIR R4
📖
5. Policy Lookup
Matches diagnosis codes and service type to applicable CMS NCD/LCD policies. CMS Coverage API integration is live and auto-updating.
policy_lookupCMS API
⚖️
6. CQL Evaluation
Executes CQL criteria against FHIR data in HAPI FHIR Clinical Reasoning. Deterministic MET/NOT_MET/UNKNOWN — not LLM inference.
cql_evaluateCQL 1.5

7. Determination
Conservative deterministic logic proposes AUTO_APPROVE, MD_REVIEW, MORE_INFO, or DENY with specific evidence citations for every criterion.
